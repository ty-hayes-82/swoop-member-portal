## (c) NEW frictions introduced by v2 fixes

1. **Chip-suppression gate is inverted for sparse members.** Both `MemberProfileDrawer.jsx:734-736` and `MemberProfilePage.jsx:418-419` suppress the small `$XXK/yr at risk` chip when `(activity.length + riskSignals.length) >= 2`. But `MemberDecayChain.buildDecayChain` triggers the demo-event fallback when `events.length < 4` (`MemberDecayChain.jsx:80`), which pushes 8 seeded events and guarantees `decayChain.length >= 2` → the chain **renders even when real events === 0 or 1**. Result: for any pilot member with 0–1 real activity/risk rows, the chain card renders its big chip AND the small gauge chip also renders. The v2 duplicate-anchor bug is only fixed for dense members. Correct gate: `score < 50 && profile.duesAnnual > 0` already guarantees the chain will render (same guard as `MemberProfilePage.jsx:438`), so the suppression should be unconditional inside that branch — or mirror the real gate `buildDecayChain(profile).decayChain.length >= 2`.

2. **`src/data/weather.js:19` still says `sunny / 73°F / wind 5` for the demo-today row (`2026-01-17`).** The v2 fix synced `teeSheet.js` and matches `cockpit.js` / `briefingService.js` on wind-advisory, but `weather.js` is the canonical week grid consumed by `WeekForecast.jsx` and `weatherService.js:204`. If the demo clicks into any weekly forecast widget, today's cell disagrees with Today / Cockpit / Tee Sheet. This is a NEW contradiction **introduced** by making tee sheet the source of truth without updating the weather data file.

3. **`PlaybooksPage.jsx:337`** shows `"Forecast: 72°F, sunny, 5mph winds"` for "This Saturday". Story 1 narrator may open Playbooks after Tee Sheet; another weather contradiction on an adjacent screen.

4. **`briefingService.js:158-160`** has `Anne Jordan time 9:14 AM / score 28` and `Robert Callahan 10:02 AM` inside a secondary `atRiskTeetimes` block, while the canonical block at `briefingService.js:48-50` (and now cockpit) say `10:15 AM / 38` and `10:42 AM / 36`. Not introduced by v2, but newly visible now that cockpit + primary briefing agree — the secondary block is the lone dissenter. Low-risk but a pedant will spot it.

## (d) NEW recommendations (not in v1 or v2)

1. **Fix the chip-suppression gate.** Replace `events >= 2` in both drawer (`MemberProfileDrawer.jsx:734-736`) and page (`MemberProfilePage.jsx:418-419`) with `buildDecayChain(profile).decayChain.length >= 2`, or — simpler — just always suppress when `score < 50 && duesAnnual > 0` (same conditions that gate the chain's render). One-line fix per file. Closes the sparse-member duplicate-anchor bug that v2 intended to close.

2. **Promote `teeSheet.weatherCondition` to a single source of truth.** Either (a) update `src/data/weather.js:19` row for `2026-01-17` to `condition: 'windy', tempHigh: 68, wind: 32`, or (b) have `teeSheetSummary` derive from `weather.js`. Same for `PlaybooksPage.jsx:337`. Eliminates two silent contradictions and future-proofs the wind beat.

3. **Reconcile `briefingService.atRiskTeetimes` secondary block.** Align `briefingService.js:158-160` times / scores with the canonical `:48-50` block so every surface agrees on James 9:20 / 42, Anne 10:15 / 38, Robert 10:42 / 36.

A shared helper prevents the next divergence and makes the suppression contract ("big chip vs. small chip") explicit in code rather than implicit in three render branches.
